API ENDPOINTS
================================================================================

AUTHENTICATION
   POST /api/auth/signup              - Register user
   POST /api/auth/callback/credentials - Login

NOTES
   GET  /api/notes                    - List all notes
   POST /api/notes                    - Create note
   GET  /api/notes/[id]              - Get specific note
   PUT  /api/notes/[id]              - Update note
   DELETE /api/notes/[id]            - Delete note

PORTFOLIO POSITIONS
   GET  /api/portfolio/positions      - List positions
   POST /api/portfolio/positions      - Add position
   PUT  /api/portfolio/positions/[id] - Update position
   DELETE /api/portfolio/positions/[id] - Delete position

WATCHLIST
   GET  /api/portfolio/watchlist      - List watchlist
   POST /api/portfolio/watchlist      - Add to watchlist
   DELETE /api/portfolio/watchlist/[id] - Remove from watchlist

CHAT
   GET  /api/chat/history             - Get conversation history
   DELETE /api/chat/history           - Clear history

DATABASE SCHEMA
================================================================================

User
   - id (PK)
   - email (unique)
   - password_hash
   - created_at, updated_at

Note
   - id (PK)
   - userId (FK)
   - title
   - content (text)
   - tags (array)
   - created_at, updated_at

NoteLink
   - id (PK)
   - sourceNoteId (FK)
   - targetNoteId (FK, nullable)
   - targetTitle (for stubs)
   - created_at

PortfolioPosition
   - id (PK)
   - userId (FK)
   - symbol
   - assetType ('stock' or 'crypto')
   - quantity (decimal)
   - avgCost (decimal)
   - created_at, updated_at

WatchlistItem
   - id (PK)
   - userId (FK)
   - symbol
   - assetType
   - created_at

ChatMessage
   - id (PK)
   - userId (FK)
   - role ('user' or 'assistant')
   - content (text)
   - created_at

SavedScreen
   - id (PK)
   - userId (FK)
   - name
   - criteria (JSON)
   - created_at, updated_at
